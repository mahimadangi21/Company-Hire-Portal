1. Introduction
1.1 Purpose
This Software Requirements Specification (SRS) document formally defines the complete functional and non-functional requirements for the Interview Management System (IMS). The document serves as the authoritative reference for design, development, testing, and stakeholder review activities.
1.2 Scope
The Interview Management System is a fully admin-controlled recruitment workflow platform designed to manage the end-to-end interview process — from resume upload through final report generation and sharing. The system eliminates any candidate-facing login or dashboard, ensuring all process control resides with the administrator.
The platform covers the following modules:
Admin Authentication
Dashboard & Analytics
Resume Upload & AI Parsing
Candidate Form Management (SMTP-based)
Video Bot Interview Management
Technical Interview Scheduling
Reports & Evaluation

1.4 System Overview
The IMS operates as a unidirectional admin-driven pipeline. Candidates never access the system dashboard. Instead, they interact only through secure, time-limited links delivered via SMTP email. The complete candidate journey is:
Resume Upload  →  AI Parsing  →  Candidate Form (Email Link)  →  Video Bot Interview (Email Link)  →  Technical Interview (Email Link)  →  Report Generation  →  Report Sharing

5. Constraints & Assumptions
5.1 Constraints
The system shall be accessible via modern web browsers (Chrome 100+, Firefox 100+, Edge 100+, Safari 15+) without requiring browser plugins.
Candidate-facing form and interview links shall function on mobile browsers without requiring the IMS application to be installed.
No candidate login, registration, or dashboard feature shall be built or implied.
Retry functionality for video bot interviews is explicitly excluded from scope.
All three monitoring controls (Camera, Microphone, Screen Sharing) for video interviews are mandatory and cannot be made optional.
The system's primary interface language shall be English.
5.2 Assumptions
The organisation operating IMS has access to a valid SMTP email server or third-party email service.
The admin user is a single individual or a small, trusted team with shared admin credentials.
Candidates have access to a modern web browser with camera, microphone, and screen-sharing capability for video interviews.
Panelists are provided meeting links through the admin's scheduling process and do not require IMS system access.
Cloud storage and computing infrastructure will be provisioned separately by the organisation.

7. Appendix — Candidate Journey Summary
The following is a textual summary of the complete admin-controlled candidate journey as implemented by the IMS:
Admin uploads one or more candidate resumes via drag-and-drop or file picker.
The system automatically parses each resume and extracts structured candidate data.
Admin reviews parsed data, makes corrections if needed, and sends the candidate a personalised form link via SMTP email.
Candidate receives the email, opens the secure link in a browser, completes the form, and submits. No IMS login required.
Admin reviews form submissions and dispatches a Video Bot Interview link via SMTP email.
Candidate opens the interview link, grants Camera, Microphone, and Screen Sharing permissions (all mandatory), and completes the AI-driven interview in a single attempt.
Admin reviews interview recordings, monitoring status, and AI-generated scores.
Admin schedules a Technical Interview, selects one or more panelists, and the system dispatches calendar invitations and meeting links via SMTP email.
Upon completion, the system aggregates all evaluation data into a structured report.
Admin reviews, finalises, and shares the report individually or with a group via email or secure link. Share status is tracked in the system.
